Relay Software Setup

Start over
Linux / VPS

 Commands in the Linux terminal are case sensitive, so make sure that the commands you write are written in the same case as listed below.

First open the terminal on your device. If it is a computer you have physical access to, there will be a program called terminal, click to launch it. If you are running a VPS, access the console. On the dashboard of your account on whatever VPS you use, there will most likely be a prompt that says "launch console".

Once you are in the terminal type in the command

sudo apt-get install tor

The terminal will ask for your password, unless you don't have one. If you do, enter it and don't mind that it looks like nothing is being typed. That's for security reasons.

If the installation is successful this will install all of the basic Tor files on your computer.

Now you need to edit the Tor configuration. The Tor configuration is located in the /etc/tor/torrc folder on your computer.

Type the command

sudo nano /etc/tor/torrc

It will ask for your password, enter it just like the last time.

If all works, you will now be editing the Tor configuration folder with a program known as Nano. Nano works slightly different than most text editors, most notably being completely contained in the command line. The commands to save, exit and more are different than most applications. On the bottom there is a list of the keyboard shortcuts that Nano uses. The ^ symbol represents the control key. If you are using a Mac keyboard on Linux you have to use the control key instead of the command key always. When you are done adding your edits you can save by pressing control + x and then answering Y to the prompt to save. 

Now you need to uncomment some lines of text on the file. Comments in code and configuration files begin with a # symbol. The # symbol indicates to the computer that the text is to be interpreted as a comment and not something to be read as a command or configuration. So we will be uncommenting (deleting the # symbol on) a few specific lines of text in this file. 

First uncomment

RunAsDaemon 1

Then uncomment

ORPort 9001

Now that this is done add your hostname into the text file. The hostname is simply the domain of the server you are hosting on the VPS. What it should look like is:

Address your.hostname.here

Then chose a nickname and enter it into the text file. For example:

Nickname Relayed

Then chose how much traffic you want a month. For example, you can input

AccountingMax 1000 GB

AccountingStart month 1 00:00

This means that you will be using only 1000 GB of data a month. Make sure to adjust this to your internet plans usage limit if you are using a computer at home to run the relay or the bandwidth limit of your VPS if you chose to use a VPS.

If you wish to add your contact info add it as so:

ContactInfo youremail@mail.com

YOU MUST DO THIS NEXT STEP. ABSOLUTELY MAKE SURE YOU HAVE THIS WRITTEN IN THE CONFIGURATION FILE. IF YOU DO NOT YOU ARE RUNNING AN EXIT NODE WHICH IS FAR OUT OF THE SCOPE OF THIS GUIDE! YOU MUST FOLLOW THIS STEP!

Make sure to write

ExitPolicy reject *:*

Now you are done. Exit Nano and make sure to save. Now enter in the terminal, not in the Tor configuration file:

systemctl enable tor
systemctl start tor 

Make sure to enter those commands in the terminal and not in the text file.

If all of these steps worked then congratulations! You have successfully set up a tor relay! You can search for your relay and metrics about it on the Tor Project's relay search page.

It is STRONGLY advised that you constantly update your system. You can update either by typing sudo apt-get update && sudo apt-get upgrade or you can enable automatic updates.

Windows

The process on Windows is very similar. There is an in-depth guide on the Tor Project website.
